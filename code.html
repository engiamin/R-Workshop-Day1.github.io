Vectors

Vectors are the simplest data structures in R. They are sequences of elements of the same basic type.

These types can be numeric, integer, complex, character, and logical. In R, the more complicated data structures are made with vectors as building-blocks.

Matrices

Matrices are the R objects in which the elements are arranged in a two-dimensional rectangular layout. They contain elements of the same types.

The basic syntax for creating a matrix in R is:

matrix(data, nrow, ncol, byrow, dimnames)

Indexing

There are multiple ways to access or replace values in vectors or other data structures. The most common approach is to use “indexing”. This is also referred to as “slicing”. In R, we use square brackets [] for indexing. Indexing in R begins at 1 (not zero like in Python).

Loops

Loops are typically used to repeat the same code a number of times for a set of cases.

In R we avoid loops wherever we can, as they tend to be slower than ‘vectorized’ computation. We also generally prefer functions like apply as this is more concise. .

There are two types of loops:

  • for-loops: repeats the same code a predefined number of times.
  • while-loops: continues until a certain condition has been met (and is therefore prone to the dreaded “infinite loop” that never finishes!)

Installing Packages

As we move through the rest of this workshop we will be working with a number of packages that need to be installed on the computer and called everytime we want to use them.

Some of the packages that we will be using:

  • openxlsx for reading, writing and editing xlsx files.
  • tidyr for data tidying.
  • ggplot2 for data visualisation.
  • dplyr for data manipulation.

The easiest way to install a package is to use the command:

install.packages(NameOfPackage)

Once we have installed packages, we must call them into R in order to use them. This must be done each time you start a new document or R session. We call packages with the following command:

library(NameOfPackage)

Getting Help in R

One of the most important aspects of learning any programming language is getting help when stuck (which will certainly happen alot!). No matter how skilled of a programmer you are, you always encounter new functions and need help understanding how to use them.

In R, the most direct ways to ask for help are:

1. The help() Function

To get online help within an R session we use the help() function. This will allow us to see the documentation on a function that we are interested in.

help(NameOfFunction)

2. The question mark (?)

?NameOfFunction

3. Internet help

There are numerous resources for finding help understanding R and creating code on the internet. My preferred method is googling the problem I am having and finding solutions on StackOverflow
